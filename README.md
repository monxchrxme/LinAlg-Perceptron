# Лабораторная работа №2. Однослойный перцептрон

Реализация алгоритма обучения однослойного перцептрона с нуля на  `NumPy` для задачи бинарной классификации. 

## 1. Архитектура модели

Перцептрон реализован как линейный классификатор. Базовое уравнение для вычисления взвешенной суммы:

$$z = Xw + b$$

Где $X$ — матрица признаков, $w$ — вектор весов, $b$ — смещение (bias). Разделяющая граница в двумерном пространстве представляет собой прямую линию: $w_1x_1 + w_2x_2 + b = 0$.

### 1.1 Binary Cross-Entropy (BCE)
По умолчанию модель обучается с использованием сигмоидной функции активации, предсказывая вероятности принадлежности к положительному классу:

$$\hat{y} = \sigma(z) = \frac{1}{1 + e^{-z}}$$

Предсказание класса выполняется по порогу `0.5`. Оптимизируемая функция потерь (Binary Cross-Entropy):

$$\mathcal{L}_{BCE} = -\frac{1}{m}\sum_{i=1}^{m}\left[y^{(i)}\log\hat{y}^{(i)} + (1-y^{(i)})\log(1-\hat{y}^{(i)})\right]$$

Формулы градиентов для mini-batch:

$$
\begin{aligned}
\frac{\partial \mathcal{L}_{BCE}}{\partial w} &= \frac{1}{m} X^T (\hat{y} - y) \\
\frac{\partial \mathcal{L}_{BCE}}{\partial b} &= \frac{1}{m}\sum_{i=1}^{m}(\hat{y}^{(i)} - y^{(i)})
\end{aligned}
$$

### 1.2 Hinge Loss (Альтернативная функция потерь)
Для режима `loss_type="hinge"` вероятности не используются. Метки классов динамически переводятся из $\{0, 1\}$ в $\{-1, 1\}$. 
Функция потерь требует, чтобы точки находились по правильную сторону от границы с некоторым запасом (margin $\ge 1$):

$$\mathcal{L}_{hinge} = \frac{1}{m}\sum_{i=1}^{m}\max(0, 1 - y_{hinge}^{(i)}z^{(i)})$$

Градиент вычисляется только для объектов, нарушающих margin ($y \cdot z < 1$). Для таких объектов:


$$\frac{\partial \mathcal{L}_{hinge}}{\partial w} = -\frac{1}{m}X_{active}^T y_{active}$$

$$\frac{\partial \mathcal{L}_{hinge}}{\partial b} = -\frac{1}{m}\sum_{i \in active} y^{(i)}$$

Если объект классифицирован верно и уверенно, его градиент равен $0$.

### 1.3 L2-регуляризация (Weight Decay)
Для предотвращения переобучения и чрезмерного роста весов реализована L2-регуляризация (управляется параметром $\lambda$):

$$\mathcal{L}_{total} = \mathcal{L}_{data} + \frac{\lambda}{2} \lVert w \rVert^2$$

Влияние на градиент по весам:

$$
\frac{\partial \mathcal{L}_{total}}{\partial w} = \frac{\partial \mathcal{L}_{data}}{\partial w} + \lambda w
$$

### 1.4 Оптимизатор (Momentum SGD)
Базовое обновление весов происходит классическим градиентным спуском. Для ускорения сходимости добавлен параметр инерции (Momentum $\beta$):

$$v_w \leftarrow \beta v_w + (1 - \beta)\nabla_w \mathcal{L}$$

$$v_b \leftarrow \beta v_b + (1 - \beta)\nabla_b \mathcal{L}$$

После этого параметры обновляются по накопленному направлению:

$$w \leftarrow w - \eta v_w$$

$$b \leftarrow b - \eta v_b$$

При $\beta = 0.0$ оптимизатор работает как классический mini-batch SGD.

## 2. Данные и предобработка

В проекте реализовано несколько способов генерации данных. Во всех случаях применяется собственная реализация `StandardScaler` (Z-score нормализация), вычисляющая $\mu$ и $\sigma$ строго по обучающей выборке во избежание утечки данных. 
Разделение на `train` и `test` (70% / 30%) осуществляется собственной функцией со **стратификацией** (сохранением баланса классов).

### 2.1 Базовый датасет
Генерируется с помощью `make_classification`: 500 объектов, 2 признака, линейно разделимые облака. Используется для тестирования основных гиперпараметров.

### 2.2 Собственный генератор сложных распределений
Разработан модуль `generators.py` с тремя типами датасетов и добавлением шума (инверсия метки с вероятностью $p=0.05$):
1. **Линейные (Гауссианы):** Два облака точек $\mathcal{N}(\mu, \Sigma)$.
2. **Нелинейные (XOR):** 4 облака по углам квадрата. Диагонали соответствуют разным классам.
3. **Нелинейные (Окружности):** Переход в полярные координаты. Внутренний круг (класс 0) и внешнее кольцо (класс 1).

### 4.2 Влияние скорости обучения (Learning Rate)
**Параметры:** epochs=100, batch_size=32, инициализация — маленькие случайные веса.  
**Варианты:** $\eta \in \{0.001, 0.01, 0.5, 1.0\}$.

* При $\eta=0.001$ функция потерь падает крайне медленно. Алгоритму нужно сильно больше 100 эпох для сходимости.
* При $\eta \ge 0.5$ график Loss начинает осциллировать (дергаться), так как шаг градиентного спуска слишком большой и перепрыгивает локальный оптимум.
* В связи с тем, что оптимизируемая функция строго выпуклая, в итоге все модели достигли дна и показали одинаковую Accuracy, однако скорость и стабильность пути кардинально различались.

### 4.3 Влияние размера батча (Batch Size)

**Параметры:** epochs=100, lr=0.1, инициализация — маленькие случайные веса.  
**Варианты:** $BS \in \{1, 16, 64, 256\}$.

* $BS=1$ (Чистый SGD): График сильно "шумит", так как направление корректируется по каждому отдельному примеру.
* $BS=256$: Линия падает очень гладко, но медленнее, так как за одну эпоху происходит всего $\approx 2$ обновления весов. Оптимум находится в диапазоне $16-64$.

### 4.4 Влияние инициализации весов
**Параметры:** epochs=100, lr=0.1, batch_size=32. 

**Варианты:** `zeros`, `small_random` ($\mathcal{N}(0, 0.01)$), `large_random` ($\mathcal{N}(0, 10)$).

Обе инициализации около нуля (`zeros` и `small`) сразу показывают плавное убывание Loss. При `large_random` модель на старте крайне уверенна в неправильных предсказаниях. График потерь стартует с огромных значений и в первые 10 эпох резко падает вниз, после чего догоняет остальные модели.

### 4.5 Обучение на нелинейных данных (Ограничения перцептрона)
**Параметры:** epochs=100, lr=0.1, batch_size=32.

**Данные:** `linear`, `xor`, `circle`.

Модель тестировалась на линейных Гауссианах, "XOR" и Вложенных окружностях.
* На Гауссианах Loss плавно сходится к нулю, разделяющая граница идеальна.
* На `XOR` и `Окружностях` перцептрон не способен сойтись: график Loss застревает на уровне $\approx 0.69$ (логарифм 2), что равноценно выдаче вероятности 0.5 для всех точек. Прямая линия физически не может разделить такие классы. Это наглядно демонстрирует потребность в многослойных нейросетях.

### 4.6 Сравнение Hinge Loss и Binary Cross-Entropy
**Параметры:** epochs=100, lr=0.1, batch_size=32.

На одном и том же датасете **Hinge Loss** показал более острую траекторию сходимости и итоговую точность `0.9128` (против `0.9060` у BCE). Это объясняется математикой Hinge Loss: она игнорирует "удобные" точки, фокусируясь на построении границы с максимальным геометрическим запасом (margin) вокруг сложных примеров.

### 4.7 Анализ влияния L2-регуляризации ($\lambda$)

**Параметры:** epochs=100, lr=0.1, batch_size=32.

Добавление штрафа за абсолютную величину весов показало следующие результаты:
* **$\lambda = 0.0$:** Обычное обучение, норма весов $\lVert w \rVert = 3.14$.
* **$\lambda = 0.01$:** Норма весов упала до $2.47$, но точность осталась на максимуме (`0.9060`). Идеальный уровень регуляризации, модель стала устойчивее.
* **$\lambda = 1.0$:** Слишком жесткий штраф. Веса "сжались" до $0.29$, модель начала недообучаться (Accuracy упала до `0.8859`).

### 4.8 Подбор гиперпараметров (Grid Search + 5-Fold Cross Validation)
**Варианты:** $\eta$ $\in$ $\{0.001, 0.01, 0.1, 0.5, 1\}$, $BS \in \{8, 16, 32, 64, 128\}$.

Для честного подбора $\eta$ и $Batch Size$ написан алгоритм K-Fold, разделяющий данные на 5 фолдов. Z-score нормализация выполнялась *строго внутри* фолдов (fit по обучающей части фолда, transform по валидационной) во избежание Data Leakage.

Анализ Heatmap показывает четкую диагональную зависимость: маленькие значения $\eta$ требуют маленького батча (чтобы делать больше шагов), а высокие $\eta$ работают лучше при больших батчах.
Лучшие гиперпараметры по среднему Accuracy: `lr=0.5, batch_size=64` (Mean Acc: $0.8820$).

### 4.9 Исследование градиентного спуска с моментом (Momentum)

**Параметры:** epochs=100, lr=0.1, batch_size=32.

Для демонстрации силы импульса, базовый шаг был занижен до `lr=0.01`.

* **SGD ($\beta=0.0$):** Алгоритм сходится долго. За 100 эпох Loss не достиг минимума.
* **Momentum ($\beta=0.9$):** Идеальная сходимость. Накопленная скорость позволяет алгоритму пролетать плоские участки. Минимум достигнут уже к 30-й эпохе.
* **Momentum ($\beta=0.99$):** Избыточная инерция. "Тяжелый" алгоритм на огромной скорости пролетел глобальный минимум и выкатился на противоположный склон (резкий скачок Loss вверх на 10-й эпохе), вызвав сильные осцилляции.
